Exploración de los delitos en la CDMX

La mayoría de las personas que vivimos en la Ciudad de México hemos sido victimas de la delinciencia o hemos escuchado a una/un amig@, vecin@ que fue afectado por la actividad criminal en la ciudad. Dichos eventos nos ha arrinconado hasta la situción en la que nos encontramos en la necesidad de evitar salir de noche, evitar lugares catalogados peligrosos, salir acompañados, voltear en todas direcciones para verificar que nadie nos esta siguiendo, hasta la creación de grupos antidelincuencia entre los vecinos para así disminuir la actividad criminal cerca de sus viviendas.

En esta ocación presentaré una exploración a los datos de las carpetas de investigación que se encuentran disponibles en la página de datos abiertos de la CDMX, aquí dejo el link de los datos. La exploración la haremos con Python y sus bibliotecas populares para el análisis de los datos como pandas y numpy.

Bien para empezar importamos las bibliotecas necesarias como pandas y numpy (bibliotecas NO librerías, library-english = biblioteca-español).

Continuando vamos a observar el nombre de las columnas y su tipo de dato:

Es importante ver qué tipo de dato tiene columna y también el significado de cada una de las variables.

  • ao_hechos: el año en que ocurrio el evento criminal.
  • mes_hechos: el mes en el cual ocurrio el evento.
  • fecha_hechos: la fecha con formato AÑO-MES-DÍA HORA:MINUTO:SEGUNDO
  • delito: una descripción breve del tipo de delito cometido.
  • categoria_delito: a qué tipo de delito pertenece.
  • fiscalia: el órgano fiscal dentro de la CDMX que se encargara de la investigación y seguimiento del delito.
  • agencia: se refiere al código perteneciente a cada agencia dentro de cada alcaldía.
  • unidad_investigacion: el equipo que se hará cargo del análisis pertinente de la situción.
  • alcaldia_hechos: la alcaldía en donde ocurrio el delito.
  • colonia_hechos: la colonia dentro de la alcaldía en la cual ocurrio el delito.
  • ao_inicio: no hay información al respecto así que supondremos que es el año en el cual empezaron las investigaciones del delito.
  • mes_inicio: de igual manera supondremos que es el mes de la iniciación de las investigaciones de los delitos.
  • fecha_inicio: la fecha de inicio de las investigaciones con formato AÑO-MES-DÍA HORA:MINUTO:SEGUNDO.
  • calle_hechos: la calle en la cual ocurrio el delito.
  • calle_hechos2: la calle colindante a la calle en la que ocurrio el delito.
  • longitud y latitud: las coordenadas geográficas referentes al lugar de ocurrencia del delito.

Podemos ver que varias de las variables deben de tener una transformación el tipo de dato, por ejemplo, ao_hechos de float a integer, fecha_hechos colocarlo en un formato datetime para que se pueda manipular con python de manera mas fácil. Pero antes de hacer las transformaciones de los datos primero verificamos la existencia de datos faltantes, para ello definimos un par de funciones:

def valores_faltantes(dataset):
    return dataset.isnull().sum(axis=0)

def grafica_barras_nulos(nulos):
    plt.figure(figsize=(10,6))
    plt.xticks(np.arange(len(nulos)), nulos.index, rotation='vertical')
    plt.ylabel('Frecuencia relativa del Total de valores perdidos en el dataset')
    plt.bar(np.arange(len(nulos)), nulos)

def formatoFecha(fecha):
    return pd.to_datetime(fecha, format='%Y-%m-%d %H:%M:%S')

def anioMesDia(fecha):
    return fecha[:10]

def mes(fecha):
    return fecha[5:7]

def dia(fecha):
    return fecha[8:10]

Para quedarnos sólo con las alcaldías de la CDMX utilizaremos otro dataset externo y nos quedaremos con la unión de ambos conjuntos, sirve que repasamos teoría de conjuntos y los join.

import re
alcaldias = pd.read_csv('alcaldias.csv', sep=',')

def quita_acentos(cadena):
    ascii_vocales_acentos = [225, 233, 237, 243, 250, 193, 201, 205, 211, 218]
    ascci_vocales         = [ 97, 101, 105, 111, 117,  65,  69,  73,  79,  85]
    indice = 0

    for letra in cadena:
        if ord(letra) in ascii_vocales_acentos:
            cadena = cadena.replace( chr(ascii_vocales_acentos[ascii_vocales_acentos.index( ord(letra))]) , 
                           chr(ascci_vocales[ascii_vocales_acentos.index( ord(letra))]))
        indice += 1
    return cadena

def limpiaCadena(cadena):
    cadena_limpia = quita_acentos(cadena)
    cadena_limpia = re.sub(r'[.]',r'',cadena_limpia)
    cadena_limpia = re.sub(r'[?|,|$|!|\\/|-|=|´|(|)|:|¿]',r' ',cadena_limpia)
    cadena_limpia = cadena_limpia.strip()
    cadena_limpia = str(cadena_limpia)
    
    return cadena_limpia.upper()

def nombre_alcaldias(alcaldia):
    return limpiaCadena(alcaldia)

alcaldias['alcaldia_hechos'] = alcaldias['NOMBRE'].apply(lambda x: nombre_alcaldias(x))

df_cdmx = alcaldias.merge(datos1, indicator = True, how='inner', on='alcaldia_hechos').loc[lambda x : x['_merge'] == 'both']

Listo!!, sólo CDMX. Ahora note que dentro de las variables delito y cat_delitos hay categorías como ROBO A NEGOCIO SIN VIOLENCIA y ROBO A NEGOCIO CON VIOLENCIA, este tipo de categorías quedarán en una sola categoría nombrada ROBO A NEGOCIO, al fin de cuentas sea o no con violencia es robo a negocio y del susto no te libras.

def delito_categoria(x):
    if 'SIN VIOLENCIA' in x:
        return x.split('SIN VIOLENCIA')[0].strip()
    elif 'CON VIOLENCIA' in x:
        return x.split('CON VIOLENCIA')[0].strip()
    else:
        return x

def delito_robo_transeunte(x):
    if 'ROBO A TRANSEUNTE' in x:
        return 'ROBO A TRANSEUNTE'
    else:
        return x

df_cdmx['cat_delito'] = df_cdmx['delito'].apply(lambda x: delito_categoria(x))
df_cdmx['cat_delito'] = df_cdmx['cat_delito'].apply(lambda x: delito_robo_transeunte(x))
